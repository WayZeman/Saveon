# Інструкція: як поставити Saveon у себе вдома на хост

Короткий покроковий гайд, щоб запустити проєкт на своєму комп’ютері або домашньому сервері (хост).

---

## Що потрібно мати

- **Комп’ютер або сервер** з доступом по мережі (домашній ПК, Raspberry Pi, VPS тощо).
- **Node.js 18 або новіший.**  
  Як перевірити: відкрий термінал (командний рядок) і введи:
  ```bash
  node -v
  ```
  Має показати щось на кшталт `v18.x.x` або `v20.x.x`.  
  Якщо команди немає — завантаж Node.js з [nodejs.org](https://nodejs.org/) (варіант LTS) і встанови.

---

## Крок 1. Отримати проєкт

- Або **скопіюй папку проєкту** (наприклад, з флешки / архіву).
- Або, якщо є **git** і посилання на репозиторій:
  ```bash
  git clone <посилання-на-репозиторій> saveon
  cd saveon
  ```

Після цього усі подальші команди виконуй **у папці проєкту** (де лежать `package.json`, папка `src`, `prisma` тощо).

---

## Крок 2. Налаштувати змінні середовища

У папці проєкту має бути файл **`.env`** з однією змінною для бази даних.

1. Якщо є файл **`.env.example`** — скопіюй його в `.env`:
   ```bash
   cp .env.example .env
   ```
2. Відкрий `.env` у будь-якому редакторі й переконайся, що там є рядок:
   ```
   DATABASE_URL="file:./dev.db"
   ```
   Це означає: база даних буде у файлі `dev.db` у папці проєкту. Для домашнього запуску цього достатньо.

Якщо хочеш зберігати базу в іншому місці (наприклад, на окремому диску), можна вказати повний шлях, наприклад:
```
DATABASE_URL="file:/home/user/data/saveon.db"
```

---

## Крок 3. Встановити залежності

У терміналі, у папці проєкту, виконай:

```bash
npm install
```

Дочекайся, поки все встановиться (може зайняти 1–2 хвилини).

---

## Крок 4. Підготувати базу даних

Один раз потрібно застосувати міграції (створити/оновити таблиці в БД):

```bash
npx prisma migrate deploy
```

Якщо хочеш одразу мати демо-користувача для перевірки:

```bash
npm run db:seed
```

Після seed можна увійти, наприклад, як `demo@family.fin` з паролем з seed (якщо він там заданий).

---

## Крок 5. Зібрати проєкт (production)

```bash
npm run build
```

Ця команда збирає додаток для запуску на хості. Виконуй її після кожного оновлення коду з репозиторію.

---

## Крок 6. Запустити додаток

```bash
npm run start
```

Сервер запуститься на порту **3000**.

- Якщо все на тому ж комп’ютері: відкрий у браузері **http://localhost:3000**
- Якщо заходиш з іншого пристрою в мережі: замість `localhost` підстав **IP-адресу** цього комп’ютера, наприклад **http://192.168.1.100:3000** (свій IP можна подивитися в налаштуваннях мережі).

Щоб зупинити сервер — у терміналі натисни **Ctrl+C**.

---

## Як запускати після перезавантаження комп’ютера

1. Зайди в папку проєкту.
2. Виконай:
   ```bash
   npm run start
   ```
   (Повторно `npm install` і `npm run build` потрібні лише якщо ти оновлював код або змінював залежності.)

Якщо хочеш, щоб додаток запускався сам після перезавантаження, можна налаштувати автозапуск (наприклад, через systemd на Linux або «Планувальник завдань» у Windows) — це вже окрема налаштування під твою ОС.

---

## Якщо щось пішло не так

- **«Port 3000 is already in use»** — порт зайнятий іншою програмою. Зупини той процес або зміни порт у скрипті `start` у `package.json` (наприклад, на 3001).
- **Помилки з базою даних** — перевір, що в `.env` правильний шлях у `DATABASE_URL` і що папка/диск існують і доступні для запису.
- **Після оновлення коду** — знову виконай `npm install` (якщо змінювався `package.json`), потім `npm run build`, далі `npx prisma migrate deploy` (якщо змінювалася схема БД), і знову `npm run start`.

Якщо напишеш, на якій системі ставиш (Windows / macOS / Linux) і яку саме помилку бачиш, можна розписати кроки точніше під твій варіант.
